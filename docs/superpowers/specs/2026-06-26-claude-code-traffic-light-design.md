# Claude Code 红绿灯 - 设计文档

**日期：** 2026-06-26
**作者：** 用户 + Codex
**状态：** 设计中

## 一、项目概述

### 1.1 项目目标
开发一个Windows桌面悬浮红绿灯程序，实时监控VSCode中Claude Code扩展的运行状态：
- 🟡 **黄灯**：Claude正在思考/处理中
- 🟢 **绿灯**：Claude正在输出结果/写入文件
- 🔴 **红灯**：需要用户确认/交互

### 1.2 适用范围
- 仅支持Windows平台
- 仅支持VSCode中的Claude Code扩展
- 支持多VSCode窗口同时运行的场景

## 二、系统架构

### 2.1 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                     Windows 桌面                        │
│  ┌──────────────┐     WebSocket      ┌──────────────┐  │
│  │  红绿灯悬浮窗 │ ◄───────────────── │  配套VSCode  │  │
│  │  (WPF/.NET)  │     localhost      │    扩展      │  │
│  └──────────────┘                    └──────────────┘  │
│                                │                        │
│                                ▼                        │
│                    ┌──────────────────────┐             │
│                    │  Claude Code官方扩展 │             │
│                    │  (源码注入状态钩子)  │             │
│                    └──────────────────────┘             │
└─────────────────────────────────────────────────────────┘
```

### 2.2 核心组件
1. **桌面红绿灯程序**：WPF .NET 8 应用，悬浮显示状态
2. **配套VSCode扩展**：负责钩子注入和状态推送
3. **WebSocket通信层**：本地进程间通信

## 三、详细设计

### 3.1 桌面红绿灯程序

#### 技术栈
- .NET 8 WPF（Windows原生性能）
- System.Net.WebSockets（WebSocket服务端）
- Win32 API（窗口置顶、拖动）

#### UI设计
- **窗口样式**：无边框、圆角、半透明背景
- **拖动**：整个窗口可拖动，支持屏幕边缘自动吸附
- **透明度**：可配置（默认80%），鼠标悬停时100%不透明
- **置顶**：始终在所有窗口最上层

#### 状态显示
| 状态 | 颜色 | 说明 |
|------|------|------|
| IDLE | 灰色 | 空闲 |
| THINKING | 黄色 🟡 | Claude正在思考/处理 |
| WRITING | 绿色 🟢 | Claude正在输出/写入文件 |
| NEEDS_CONFIRM | 红色 🔴 | 需要用户确认（闪烁3次后常亮） |

#### 气泡消息
- 状态变化时显示1-2行简短描述
- 例如："需要确认计划"、"正在写入文件"

#### 右键菜单
- 透明度设置（滑块 0-100%）
- 红灯闪烁次数设置
- 轮播间隔设置
- 屏幕边缘吸附开关
- 退出程序

#### 多窗口轮播逻辑
- 维护活跃Claude实例的Map
- 默认每隔3秒轮播显示不同窗口的状态
- **优先级规则**：任何一个实例是🔴红灯时，优先显示红灯，不轮播直到用户处理完

### 3.2 VSCode配套扩展

#### 核心功能
1. **钩子注入器**：自动检测Claude Code扩展安装位置，注入状态监控钩子
2. **状态采集**：从Claude Code内部捕获精确状态变化
3. **WebSocket客户端**：连接到红绿灯程序，实时推送状态
4. **一键重注入**：Claude Code更新后，提供按钮重新注入钩子

#### 状态定义
```typescript
enum ClaudeState {
  IDLE = ''idle'',              // 空闲
  THINKING = ''thinking'',      // 思考中 🟡
  WRITING = ''writing'',        // 输出/写入 🟢
  NEEDS_CONFIRM = ''needs_confirm''  // 需要确认 🔴
}
```

#### 注入实现
- 定位Claude Code扩展的编译后JS文件（`extension.js`）
- 在状态机/UI渲染关键节点注入钩子函数
- 钩子调用配套扩展暴露的API，推送状态

### 3.3 通信协议

#### WebSocket连接
- **地址**：`ws://localhost:19876`（端口可配置）
- **服务端**：红绿灯程序
- **客户端**：VSCode配套扩展

#### 消息格式
```typescript
// 状态推送消息
{
  "type": "state_change",
  "payload": {
    "state": "thinking" | "writing" | "needs_confirm" | "idle",
    "message": "思考中...",           // 气泡显示文字
    "vscodeWindowId": "window-uuid", // 窗口唯一标识
    "timestamp": 1234567890          // 时间戳
  }
}

// 心跳消息（每30秒）
{ "type": "ping" }

// 连接响应
{ "type": "pong" }
```

### 3.4 配置项

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| opacity | number | 80 | 窗口透明度 (0-100) |
| redFlashCount | number | 3 | 红灯闪烁次数 |
| carouselInterval | number | 3000 | 多窗口轮播间隔（毫秒） |
| edgeSnap | boolean | true | 屏幕边缘吸附 |
| autoStart | boolean | false | 开机自启动 |
| websocketPort | number | 19876 | WebSocket端口 |

## 四、错误处理

### 4.1 连接异常
- VSCode扩展断开连接时，红绿灯显示灰色（离线）
- 自动重连机制（指数退避）
- 连接状态在气泡中提示

### 4.2 钩子注入失败
- VSCode扩展中显示错误提示
- 提供手动选择注入路径的功能
- 提示用户Claude Code可能已更新，API不兼容

### 4.3 Claude Code更新
- 检测到Claude Code版本变化时，提示重新注入
- 提供一键重新注入按钮

## 六、验收标准

1. 红绿灯悬浮窗能正常显示，可拖动到任意位置
2. Claude Code思考时亮黄灯，输出时亮绿灯
3. 出现Accept确认框时亮红灯，闪烁3次后常亮
4. 多VSCode窗口时，状态正确轮播
5. 红灯状态优先显示，不轮播
6. 透明度可配置，右键可退出程序
